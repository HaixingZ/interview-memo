# interview-memo（访谈纪要 Skill）

这是一个用于把**访谈/会议原始记录**整理成**结构化、咨询风格纪要**的 Skill，重点适配 TMT / 互联网 / 广告技术（AdTech）等话题

## 默认语言

本 Skill 的输出内容默认使用**简体中文**撰写

## 适用输入

- 逐字稿/录音转写（可包含口头禅、打断、重复、跑题）
- 会议记录（可能多人对话、信息碎片化）

建议你在输入里补充：访谈对象身份、公司/赛道、访谈目的（为了建模、竞品分析、策略判断等），这样纪要更贴近你的决策问题

## 输出结构（你会得到什么）

- 结构化纪要正文（按模块归类，而不是按对话顺序复述）
- 关键事实与指标（GTV/ROI/CPM/转化率/时间点等）
- 逻辑链路与关键假设（可用于后续建模/写报告）
- `[分析师注]`：对信息缺口、矛盾点、需要追问的地方做标注

## 工作流程（Skill 的处理步骤）

### Phase 1：对齐目标与大纲（Pre-work）

1. 识别核心主题：明确这次访谈的主问题是什么（例如“抖音 vs 美团”“线索广告投放链路”“某行业 UE 结构”）
2. 拟定纪要大纲：先把可能的分析框架搭出来（例如“流量漏斗 → 转化 → UE 模型”“竞争格局 → 运营打法 → 盈利结构”）
3. **强制确认**：把大纲给你确认，**在你确认前不输出完整纪要**（避免方向跑偏）

### Phase 2：清洗与要点抽取（Processing）

1. 去噪：删除口水话、寒暄、跑题段落，只保留“可用于判断/建模”的信息
2. 标准化表述：对模糊说法尽量转成范围或条件描述，并保留原话里的置信度词（如“约”“保守”“大概”）
3. 抽取硬信息：把所有“数字 + 口径 + 时间”的信息单独抽出来，形成 Key Metrics Snapshot 的素材

### Phase 3：重组逻辑与结构化表达（Analysis）

1. 逻辑重建：把碎片信息按业务逻辑重新排列，例如拆解 `收入 = 流量 × 广告加载 × CPM`、或 `利润 = 收入 - 变动成本 - 固定成本`
2. 对比结构：涉及多平台/多年度时，用表格做横向/纵向对比（例如 2023 vs 2024、平台 A vs 平台 B）
3. MECE 归类：把观点按互斥且穷尽的方式组织（宏观/运营/财务/竞争等），避免重复

### Phase 4：洞察标注与自检（Finalizing）

1. `[分析师注]`：对关键结论补上推导、指出缺口、给出下一步追问方向
2. 一致性校验：检查口径是否自洽（例如利润率是否能由收入/成本拆出来，时间线是否矛盾），发现冲突就显式标红或提示

## 使用方式（建议）

1. 直接粘贴逐字稿/转写文本
2. 在开头补 3 行信息：
   - 访谈对象：
   - 访谈主题/目的：
   - 你最关心的 3 个问题：
3. 我会先给你一份“拟定大纲”让你确认，你确认后再输出完整纪要

## 注意事项

- 本 Skill 不会凭空编造数据；缺失信息会用 `[信息缺口]` 或 `[分析师注]` 标出来
- 如果你的原始记录涉及敏感信息，建议在粘贴前先做脱敏（公司名/人名/具体金额口径等）
